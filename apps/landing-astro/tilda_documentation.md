## КАК РАБОТАЕТ TILDA

### Основная структура

- **Бизнес-план**: до 5 сайтов с собственным доменом у каждого
- **Free/Personal план**: 1 сайт
- Один сайт = несколько страниц (портфолио, блог, магазин) или одна страница (лендинг)
- У каждого сайта: свои настройки стилей (шрифты, цвета) и домен

### Блоки — основной элемент Tilda

- Страница создаётся путём добавления и расстановки блоков
- Более **550 готовых блоков** (покрывают большинство комбинаций)
- Все блоки настраиваемые — не нужно создавать с нуля
- Можно добавлять контент прямо на странице: кликнуть на текст, перетащить изображение
- Оранжевая кнопка блока → Content panel → доступ к дополнительным настройкам

### Работа с блоками

- Блоки можно: дублировать, перемещать, вырезать, копировать, вставлять
- Блок можно вставить: на текущей странице, другой странице, другом сайте в том же аккаунте
- **Preview** — предпросмотр страницы перед публикацией
- Tilda сохраняет изменения автоматически
- После публикации страницы — доступна ссылка

### Типы блоков по категориям (из Block Library)

- **Cover** — обложки / первые экраны
- **Form** — формы сбора данных
- **Store** — блоки для магазина
- **Other** — утилитарные блоки (T123 = HTML embed, T173 = Anchor, T178 = Smooth Scroll, T875 = copy protection, T1093 = popup modifier)
- **Gallery**, **Contacts**, **Footer**, **Menu** и другие стандартные категории

## ZERO BLOCK

Zero Block — встроенный профессиональный редактор для создания уникальных блоков любого дизайна.

### Добавление Zero Block

- Кнопка "Zero" в самом низу страницы
- Или найти в Block Library → категория "Other" → самый нижний

Zero Block сохраняет все возможности обычного блока: копирование, удаление, скрытие, настройки. Вместо кнопки "Content" — кнопка "Block Editor".

### Интерфейс Block Editor

Два рабочих пространства (контейнера):

1. **Grid Container** — основное рабочее пространство, 12 колонок = 1200 px. Всегда по центру. Элементы внутри остаются в 12-колоночной сетке при любом размере экрана.
2. **Window Container** — обозначает границы браузерного окна

### Добавление и редактирование элементов

**Кнопка "+" (верхний левый угол Block Editor)** → добавить элемент:
- Text
- Image
- Shape
- Button
- Video
- Tooltip
- HTML
- Form
- Gallery
- Vector

Также можно перетащить изображение из папки прямо в блок.

### Управление мышью

- Перемещение, ресайз, копирование (Alt + drag), мультиселект (Shift)
- Resize изображений без изменения пропорций
- Высота текстовых элементов меняется автоматически по объёму текста
- Кнопки и фигуры ресайзятся в любом направлении

### Горячие клавиши (shortcuts)

| Действие | Клавиши |
|---|---|
| Переключение типа экрана | ⌘+1...5 |
| Скрыть сетку | G |
| Скрыть настройки | TAB |
| Скрыть элементы управления | F |
| Отменить действие | ⌘+Z |
| Сохранить | ⌘+S |
| Копировать | ⌘+C |
| Вставить | ⌘+V |
| Переместить (Shift = шаг) | (Shift+) ←↑→↓ |
| Удалить | Backspace |
| Изменить прозрачность | 0...9 |
| Заблокировать элемент | L |
| Размер шрифта | +/– |
| Межстрочный интервал | ⌘+↑/↓ |
| Межбуквенный интервал | ⌘+←/→ |
| Выбрать все | ⌘+A |
| Слои: скрыть/показать | ⌘+L |
| Гайд горизонтальный | ⌘+H |
| Гайд вертикальный | ⌘+Shift+L |

*На Windows: Ctrl вместо ⌘*

### Панель настроек элементов (TAB)

Верхний ряд кнопок → быстрое выравнивание по горизонтали/вертикали относительно контейнера.

#### Текстовый элемент
- Полные настройки шрифта
- Изменение размера: клавиши –/+
- Межстрочный: ⌘+↑/↓
- Межбуквенный: ⌘+←/→
- Копировать: Alt + drag
- Прозрачность: клавиши 0...9
- Ссылка на элементе

#### Изображение
- После загрузки → размер = размер фрейма
- "Original size" в Settings → сброс до оригинального размера
- Радиус скругления → обрезать в круг
- Тень: цвет, прозрачность, смещение X/Y, блур, spread
- Alt tag (SEO) для каждого изображения
- Изображение → ссылка (открыть в том же / новом окне)

#### Фигура (Shape)
- Прямоугольник: менять стороны через контрольные точки или значения
- Круг: радиус скругления = половина стороны квадрата
- Линия: высота 1-5 px
- Фигуры: тень, рамка, ссылка

#### Кнопка (Button)
- Размер, цвет, радиус скругления
- Тень, рамка
- Текст, ссылка, прозрачность, цвет шрифта
- Поведение на hover: цвет фона, цвет текста, цвет рамки

#### Tooltip
- Интерактивный элемент с подсказкой при hover
- Настройки: цвет, тень, размер, иконка, изображение, анимация
- Важно: проверить расположение на всех разрешениях (чтобы tooltip не обрезался)

#### Форма в Zero Block
- Горизонтальная или вертикальная форма
- Все базовые настройки форм: поля, подключение сервисов, success/error сообщения, дизайн, анимация
- Input fields → настройка полей
- Services → подключение сервисов сбора данных
- Переменные в имени шагов формы → `{{form.name=def}}`

#### Галерея в Zero Block
- Stretch: Cover (заполняет область, может обрезать) / Contain (оригинальный формат, не обрезает)
- Position: Left Top, Center, Right Bottom и т.д.
- Loop: Loop (бесконечно) / None
- Slide Speed: None / Slow / Fast
- Autoplay: скорость автосмены в секундах
- Zoomable: Zoom on click / None
- Arrows и Dots — вид навигации
- Для каждого слайда: подпись, Alt tag, YouTube/Vimeo видео, ссылка

### Artboard Settings (клик на пустое место)

- Высота Grid Container в пикселях (или drag края контейнера)
- Ширина Grid Container фиксирована: 1200px
- Цвет фона всего блока
- Фоновое изображение всего блока
- Filter Start/Filter End — фильтры для фонового изображения
- `Fixed` — содержимое движется при скролле, изображение стоит на месте (parallax effect)

### Zero Block как Pop-up

1. Добавить блок-модификатор **T1093** из категории "Other"
2. В Content tab T1093: указать ID нужного Zero Block (или выбрать через "Choose Zero Block")
3. По умолчанию ссылка для открытия popup: `#zeropopup` (можно изменить, сохранив `#`)
4. Блок будет **скрыт** на странице и откроется как обычный popup
5. В настройках T1093: фон (цвет, прозрачность), цвет иконки закрытия, скорость анимации, z-index и другие

**z-index** — порядок перекрытия блоков на странице. Числовые значения (например, блок с z-index 10 перекрывает блок с z-index 5).

## ZERO BLOCK: RESPONSIVE DESIGN

### Grid Container на оси X

- Ширина Grid Container всегда фиксирована и центрирована
- При переключении типа экрана ширина меняется: 980, 640, 480, 320px — остаётся фиксированной и центрированной
- Высота Grid Container = высота блока

### Точки останова экранов

- 1200+ px — десктоп (max)
- 980-1200 px — горизонтальный планшет
- 640-980 px — вертикальный планшет
- 480-640 px — горизонтальный смартфон
- 320-480 px — вертикальный смартфон

### Window Container на оси Y (100% экрана)

Чтобы Zero Block занимал 100% высоты экрана:
1. Window Container height = 100% в Artboard Settings
2. Затем выбрать выравнивание Grid Container: Top / Center / Bottom / Stretch (по умолчанию: Center)

### Контейнер и Origin элемента

**По умолчанию**: все элементы прикреплены к Grid Container, origin в левом верхнем углу.

**Смена контейнера** (например, логотип всегда в левом верхнем углу):
1. Открыть Settings panel элемента
2. Изменить тип контейнера на **Window Container**
3. Origin сместится к верхнему левому углу Window Container

**Origin** (координатная точка) — 9 вариантов расположения: right top, bottom left, center center и т.д.

Чтобы изменить origin → открыть раздел `+Container` в Settings panel.

### Координаты элемента

- X и Y — позиция относительно origin в контейнере
- Можно задавать в пикселях или **процентах** (для fluid/резиновых элементов)

### Fluid-элементы (% вместо px)

Пример: правая половина экрана всегда жёлтая:
- Форма: Container = Window Container
- Ширина = 50%, Высота = 100%
- X-offset = 50%

### Auto Scale

Масштабирование Grid Container пропорционально ширине окна браузера:

**Site Settings → Artboard Settings → Scale Grid Container → Autoscale to Window Width**

- Grid Container растягивается на всю ширину экрана
- Все элементы (в Grid Container) масштабируются пропорционально
- Работает только для элементов в Grid Container (не Window Container)
- Высота блока динамическая — меняется пропорционально ширине
- Изображения: для иконок — SVG, для фото — загружать больше нужного (max 1920px)

### Пример: fullscreen cover

1. Background image в Artboard Settings + Window Container height = 100%
2. Заголовок, подзаголовок, кнопка: origin = Center Center
3. Стрелка вниз: Container = Window Container, origin = Center Bottom, Y-offset = -70px

## ZERO BLOCK: АНИМАЦИЯ (БАЗОВАЯ)

Анимация применяется к любому элементу Zero Block.

### Типы анимации

1. **Базовая анимация** — простые готовые эффекты
2. **Пошаговая анимация** — сложные кастомные эффекты

### Базовые эффекты

- **Fade In** — появление через прозрачность
- **Fade In Right / Left / Up / Down** — появление со стороны
- **Zoom In** — увеличение

**Общие настройки для всех эффектов:**
- **Duration** — длительность анимации в секундах
- **Delay** — задержка перед стартом в секундах
- **Trigger Offset** — смещение в пикселях от нижнего края Window Container для начала воспроизведения

**Distance** (для Fade In Right/Left/Up/Down) — расстояние, которое проходит элемент от точки появления.

**Zoom In → Scale**: < 100% = элемент увеличивается до оригинала; > 100% = элемент уменьшается и принимает исходный размер.

### Parallax

**На скролл:**
- Speed < 100%: элемент движется медленнее прокрутки страницы
- Speed > 100%: элемент движется быстрее прокрутки

**На движение мыши:**
- X и Y — максимальное смещение при параллакс-наклоне

### Fixing (фиксация)

Фиксация элемента при прокрутке:
- **On Window Top / Center / Bottom** — область экрана, где начинается фиксация
- **Trigger Offset** — точка фиксации относительно выбранной области
- **Distance** — расстояние в пикселях, которое проходит элемент во время фиксации

## ZERO BLOCK: ПОШАГОВАЯ АНИМАЦИЯ

### Режимы анимации

1. **Базовая анимация** — готовые эффекты, 90% случаев
2. **Step-by-step анимация** — для сложных, творческих идей

### Начало работы

1. Добавить Zero Block на страницу
2. Кликнуть "Edit Block"
3. Добавить элемент, выбрать его
4. Открыть Settings panel → раздел "Step by Step Animation" → кнопка "Add"

**Важно**: добавление пошаговой анимации отменяет базовую анимацию для этого элемента.

### События запуска (Events)

- **Element on Screen** — анимация запускается, когда элемент появляется на экране
- **Block on Screen** — анимация запускается, когда весь блок появляется на экране
- **On Scroll** — анимация воспроизводится во время скролла
- **On Hover** — анимация запускается при наведении курсора
- **On Click** — анимация запускается при клике

### Дополнительные опции старта

- **Start Trigger** — зона: Window Top / Window Center / Window Bottom
- **Trigger Offset** — смещение точки старта
- **Loop** — зацикливание анимации

**Play Element** — тест анимации одного элемента  
**Play All** — тест анимации всех элементов блока

### Шаги анимации

- Первый шаг "Start" — по умолчанию
- Кнопка "Add step" → добавить шаг
- Синяя рамка элемента — оригинальная позиция
- Зелёная рамка — редактируемый шаг

**Параметры каждого шага:**
- **Duration** — длительность шага в секундах
- **Move** — координаты перемещения относительно оригинальной позиции
- **Scale** — масштаб к концу шага
- **Opacity** — прозрачность к концу шага
- **Rotate** — поворот в градусах к концу шага
- **Easing** — тип кривой анимации:
  - Linear — линейное
  - easeIn — замедление в начале
  - easeOut — замедление в конце
  - easeInOut — замедление в начале и конце
  - bounceFin — отскок в конце
- **Delay** — пауза перед началом шага

### Примеры использования

- Элементы появляются и движутся при скролле
- Анимация при hover или click
- Бегущая строка (looping ticker)
- Сборка элемента из фрагментов
- Зацикленная анимация (ship, пузырьки)
- Несколько элементов движутся в разных направлениях
- Элементы движутся вдоль экрана при скролле
- Анимированный текст
- Скрытие элементов при скролле
- Параллакс-эффект

## ВСТАВКА HTML-КОДА

### Блок T123 ("Embed HTML Code")

**Добавить**: Block Library → Other → T123

- Поддерживает: HTML, JavaScript (внутри `<script>`), CSS (внутри `<style>`)
- **PHP не поддерживается** (можно разместить PHP на своём сервере и слать данные через Webhook)
- В режиме редактирования и предпросмотра — отображается как текст
- Для работы — опубликовать страницу

### HTML в Zero Block

HTML-элемент в Zero Block аналогичен блоку T123 — без существенных отличий.

### HTML в <head> сайта или страницы

**Для всего сайта:** Site Settings → More → HTML code for the head section

**Для конкретной страницы:** Page Settings → Additional → HTML code for the head section

> **Важно**: поддержка по вопросам своего кода не предоставляется. Сторонние решения могут перестать работать из-за изменений в сторонних сервисах или новых фичей Tilda. При ошибках — отключить код-блок и обратиться к автору кода.

## TILDA API

API используется для синхронизации контента Tilda с вашим сайтом.

### Основные правила

- Все запросы: метод **GET**, ответы в **JSON**
- Лимит: **150 запросов в час**
- Контент должен сохраняться на вашем сервере (изображения, скрипты, стили) — не проксировать Tilda для каждого посетителя
- Типичная синхронизация: 1-3 запроса
- API доступен только для **Tilda Business Plan**

### Получение ключей

Site Settings → Export → API Integration → Public Key и Secret Key

### Базовый URL

```
https://api.tildacdn.info
```

### Список запросов

#### Список проектов

```
GET https://api.tildacdn.info/v1/getprojectslist/?publickey=XXX&secretkey=XXX
```

**Ответ:**
```json
{
  "status": "FOUND",
  "result": [
    { "id": "0", "title": "First Project", "descr": "Some info" },
    { "id": "1", "title": "Second Project", "descr": "" }
  ]
}
```

#### Информация о проекте

```
GET https://api.tildacdn.info/v1/getprojectinfo/?publickey=XXX&secretkey=XXX&projectid=0
```

**Ответ включает:**
- `id`, `title`, `descr`
- `customdomain`, `export_csspath`, `export_jspath`, `export_imgpath`
- `indexpageid`, `favicon`, `page404id`
- `images[]` — массив с `from` и `to` (пути к изображениям)

Параметр `webconfig=htaccess` или `webconfig=nginx` → образец конфигурации веб-сервера.

#### Список страниц проекта

```
GET https://api.tildacdn.info/v1/getpageslist/?publickey=XXX&secretkey=XXX&projectid=0
```

**Каждая страница:**
- `id`, `projectid`, `title`, `descr`, `img`, `featureimg`
- `alias`, `date`, `sort`, `published`, `filename`

#### Информация о странице + body HTML

```
GET https://api.tildacdn.info/v1/getpage/?publickey=XXX&secretkey=XXX&pageid=1001
```

Возвращает: все поля страницы + `html` (body код), `js[]`, `css[]`

#### Информация о странице + full HTML

```
GET https://api.tildacdn.info/v1/getpagefull/?publickey=XXX&secretkey=XXX&pageid=1001
```

Возвращает: все поля + `html` (полный HTML страницы)

#### Страница для экспорта + body HTML

```
GET https://api.tildacdn.info/v1/getpageexport/?publickey=XXX&secretkey=XXX&pageid=1001
```

Возвращает: все поля + `images[]`, `js[]` (с attrs: async/defer/nomodule), `html` (с локальными путями), `filename`

> **Важно**: При каждом запросе рекомендуется скачивать и обновлять статические файлы (скрипты и стили) — они постоянно меняются, и через API нельзя проверить актуальность.

#### Страница для экспорта + full HTML

```
GET https://api.tildacdn.info/v1/getpagefullexport/?publickey=XXX&secretkey=XXX&pageid=1001
```

Возвращает: все поля + `images[]`, `html` (полный HTML с локальными путями)

### Webhook для автосинхронизации

Site Settings → Export → API Integration → укажите URL скрипта.

При каждом клике "Publish" → GET-запрос на ваш URL с параметрами: `pageid`, `projectid`, `published`, `publickey`

**Ваш webhook должен ответить строкой `"ok"`** (PHP: `echo "ok";`)

- При отсутствии ответа — 2 повторных попытки (3 всего)
- Задержка до 1-20 минут после публикации

## WEBHOOK

### Настройка

1. Site Settings → Forms → Webhook
2. Ввести URL скрипта
3. Опционально: поставить галочку "Send cookies"
4. Сохранить
5. В Content panel блока с формой → выбрать чекбокс "WEBHOOK" → Save and close → Опубликовать

**Требования к скрипту:**
- Должен быть доступен
- Отвечать в течение **5 секунд**
- Использовать **HTTPS**
- При неответе — 2 повторные попытки (всего 3)

Данные передаются через **POST**.

### PHP пример: отправка на email

```php
<?php
header('Access-Control-Allow-Origin: *');

$headers = "From: from@webhookservesite.com";
$message = print_r($_POST, true);
@mail('to@mail.com', 'Tilda TEST', $message, $headers);

echo "ok";
?>
```

### Структура POST-данных

```
Email=test%40email.com
Name=test+name
Phone=0123456789
Comments=example+comment
Selectbox=red
Checkbox=yes
Date=25-03-2017
Time=17%3A59
Quantity=124
Url=website.com
tranid=467251%3A8442970    # Lead ID
formid=form48844953        # Block ID
```

- `@` кодируется как `%40`
- `:` кодируется как `%3a`
- Декодирование через `urldecode()`

### Получение адреса отправки

```php
$_SERVER["HTTP_REFERER"]
```

### Cookie (если включено "Send cookies")

```
COOKIES=_ga%3DGA1.2.1861016115.1519204131%3B...
```

UTM-параметры попадают в cookies если пользователь заходил с UTM менее 30 дней назад.

### Дополнительные настройки Webhook

В Site Settings → Forms → Webhook:
- **Передача данных о товарах как массивы**
- **Передача externalid** (внешний код) из Product Catalog — только при включённых массивах

### Инструменты тестирования

- http://requestcatcher.com
- https://webhook.site

### Получение конкретного поля

```php
$_POST["Date"]  // где "Date" — имя переменной поля
```

Если имя переменной изменено:
```php
$customnamefield=yellow
```

3. Пользователь заходит через: `yoursite.com/members/login`

### Страница регистрации (Signup)

Создаётся автоматически, включается в Members Area Settings.

URL: `yoursite.com/members/signup`

### Дизайн страниц входа/регистрации

Members Area Settings → Advanced → Login & Signup Page Settings → Edit

**Настройки:**
- Form Style: центрированная форма или 2 колонки
- Upload Image: фон страницы, фон формы, градиент, паттерн
- Typography: шрифты для заголовков и текста
- Form Input Style: цвета полей, иконок, рамок
- Buttons: цвета кнопки + hover

### Методы входа и регистрации

Members Area Settings → Advanced → Signup & Login Methods:

- **По email** с постоянным паролем ИЛИ одноразовым кодом
- **По телефону** с одноразовым кодом (требует настройки SMS-шлюза)
- **По email ИЛИ телефону** — пользователь выбирает метод

Опция "Login and registration in one form" — вход и регистрация в одной форме.

Опция "Prevent login change" — запрет изменения email/телефона.

### Форма регистрации (Custom Signup)

Использование блока формы для регистрации через сбор данных:

1. Site Settings → Forms → активировать "Members Area" data capture tool
2. Выбрать группу для добавления новых участников
3. Добавить блок с формой → Content panel → выбрать "Members Area" → Save → опубликовать

**Важно при нескольких формах:** добавить скрытое поле с уникальным значением в каждую форму (например, название курса) чтобы Tilda не фильтровала дубликаты.

### Платная регистрация (доступ к контенту)

1. Добавить блок с корзиной + платёжной системой
2. Подключить "Members Area" к форме в корзине → выбрать группу
3. Site Settings → Payment Systems → "Send info to data collection services only after payment"
4. Group Settings → "Approve membership requests without confirmation"
5. Поле "Expected payment for automatic entry" → цена для верификации

**Поток платёжной регистрации:**
- Клик "Купить" → открывается корзина → заполнение формы → оплата → Tilda получает подтверждение → создаётся аккаунт → письмо с данными → пользователь входит

### Продажа доступа к нескольким группам/курсам через корзину

*(Beta функция)*

1. Group Settings → скопировать public ID группы (снизу)
2. Product Catalog → карточка товара → поле "Product external ID" → вставить ID группы
3. Site Settings → Forms → Members Area → "Receive group id from Product Catalog"
4. В блоке корзины (ST100) → Content tab → выбрать чекбокс Members Area

### Продажа доступа через ссылку

*(Beta функция)*

```
#order:Название продукта =2000:::extid=000000000
```

Где `000000000` — public ID группы.

С изображением в корзине:
```
#order:Название =2000:::image=URL:::extid=000000000
```

### Управление группами

- Divide members into groups
- Для каждой группы: список страниц с доступом + index-страница
- Create new group → Enter title → Create new group

**Вкладки группы:**
- **Members** — список участников (добавить/удалить/переместить)
- **Pages** — переключатель "Include" → страница доступна только этой группе

**Одновременный вход:** не более 3 устройств с одним логином.

### Автоматическое удаление из группы

Group Settings → Group Access Period:
- Период от 1 дня до 1 года
- Конкретная дата
- Применить ко всем уже добавленным участникам

Индивидуальное управление сроком: открыть группу/All Members → найти пользователя → кликнуть на дату → изменить.

### Стартовая страница (Start Page)

**Одна группа:** после входа → index-страница группы (настраивается в Group Settings → Index Page)

**Несколько групп:** Members Area Settings → Advanced → Start Page Settings → настроить:
- Тексты над контейнером
- Фон (изображение или цвет)
- Дизайн контейнера и типографика

### Email-шаблоны

Members Area Settings → Mail Settings → Edit

Переменные для персонализации: `{{login}}`, `{{password}}`, URL страницы входа, домен.

Типы уведомлений: регистрация, зачисление в группу, смена пароля, изменение email.

### Защита контента от копирования

**Текст:** Добавить блок **T875** из категории "Other" на страницу (запрещает выделение текста и правый клик)

**Видео:** использовать сервисы с защитой (Kinescope, Boomstream) + вставить через HTML блок или HTML в Zero Block

## ТЕХНИЧЕСКИЕ ОГРАНИЧЕНИЯ И ВАЖНЫЕ ДЕТАЛИ

### Изображения

- Максимальный размер загружаемого изображения: **1680px** по ширине (по умолчанию)
- Для 1920px: при загрузке кликнуть иконку шестерёнки → "Enable 1920px wide image uploading"
- Изображения > 1920px не поддерживаются (влияет на производительность)
- Для иконок в Zero Block: рекомендуется SVG
- Для фото: загружать больше необходимого для корректного масштабирования

### Блоки

- Каждый блок имеет уникальный ID (`#rec129000`)
- ID находится в нижней части Settings panel блока
- T178 (smooth scroll) — **только один** на страницу
- Z-index кастомных блоков — настраивается для управления перекрытием

### Публикация

- Tilda автоматически сохраняет изменения
- Страницу нужно **опубликовать** для вступления в силу изменений
- robots.txt и sitemap.xml генерируются автоматически
- После смены favicon — очистить кэш браузера

### Домен и SSL

- Free SSL для сайтов на Tilda
- При экспорте кода — SSL отключается, нужен сторонний сертификат
- Подключение собственного домена делает его canonical автоматически

- Email: team@tilda.cc
- Форма обратной связи в аккаунте Tilda
- Документация по бухгалтерии: docs@tilda.cc
- Поддержка не помогает с вопросами по пользовательскому коду
